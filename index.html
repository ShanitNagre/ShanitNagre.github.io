How I think

Operating principles

Written the hard way — from things that broke, teams that reverted, and numbers that didn't survive scrutiny.

01
Architecture

LLMs for judgment, deterministic systems for facts

The model gets to interpret, classify, and recommend. It never gets the final word on a dollar amount, a code, or a date. Validation gates, not vibes.

02
Metrics

Number → denominator → definition → what moved it

Every metric I report survives four questions. If a number can't answer all four, it doesn't go in the dashboard — or the deck.

03
Quality

Instrument before you launch

Eval loops go live before the model does: golden datasets, precision, escalation rate, regression tests on every change. If you can't measure it on day one, it isn't ready.

04
Adoption

Kill the parallel old path

Trust is built in small doses — the claims pipeline ran 200 claims a week before touching the full queue. But once the numbers hold, the old process gets retired, or teams quietly revert.
