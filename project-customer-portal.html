PROBLEM & APPROACH

🔥 The Problem
Avanse borrowers had no self-serve channel. Every question — repayment schedule, NOC request, interest certificate — required a support call. Support was overwhelmed. Customers were frustrated. And the ops team was manually sending documents that should have been available in an app.

There was no customer portal at all. We were building from zero.
💡 The Approach
Mobile-first, loan-lifecycle app. Onboarding with DigiLocker + PAN + biometric APIs. Personalised dashboard showing repayment schedules, outstanding dues, document vault. Firebase-triggered notifications for due reminders and loan status.

The design principle: every interaction that currently requires a support call should be self-serviceable in under 3 taps. We validated every flow with 50+ real users before MVP.
Delivery

10 WEEKS TO V1

W1–2
Discovery & Research
50+ user interviews, support ticket analysis, competitive teardown of 6 lending apps. Defined MVP scope.
W3–4
Wireframes & API Spec
Figma wireframes for all 12 core flows. RESTful API specs for onboarding, repayment, and document modules.
W5–6
Usability Testing
Maze testing with 50 users. Task completion rate: 72% → iterated 3 flows. Improved to 94% before handoff.
W7–9
Build & Integration
Engineering sprint. DigiLocker, Aadhaar, PAN API integration. Firebase notifications. Mixpanel instrumentation.
W10
Soft Launch & Monitoring
Rolled out to 500 users. Monitored funnel drops, escalation rate, CSAT daily. Task completion hit 96%.
